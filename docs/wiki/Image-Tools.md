# 🖼️ 图像工具集 (Image Tools)

Auto-Feed Refactor 集成了强大的图片处理与转存引擎，完全继承并增强了原版功能。

---

## 1. 核心功能概览

### ✅ 单图/多图转存 (Image Re-hosting)
- **支持源**: 任意图片链接（包括带有防盗链保护的图片）。
- **目标图床**: PTPIMG, PIXhost, IMGBOX, FreeImage, Gifyu, Hostik。
- **批量操作**: 自动扫描页面上的所有截图，勾选后一键批量上传到 PIXhost，并智能生成缩略图 BBCode。

### ✅ 图片处理 (Image Processing)
可通过点击面板上的 `图片处理/Image Tools` 大按钮打开独立窗口。
- **提取**: 从大段简介文本中提取所有图片链接。
- **替换**: 批量替换文本中的图片域名或路径。
- **对比图**: 快速制作带有 Label 的 Screenshot Comparison。
- **缩略**: 自动生成缩略图代码。

### ✅ 图床桥接 (Bridge)
针对 HDBits, IMGBOX, Hostik 等无法直接通过 API 上传的图床，脚本使用独创的"桥接"技术：
1. 在源页面点击对应图床图标。
2. 脚本自动跳转到图床上传页。
3. 自动将源图片下载并填入文件选择框。
4. 你只需点击 "Start Upload"。

---

## 2. 详细使用指南

### A. 批量截图转存
在资源详情页，脚本会自动识别 Description 中的截图区域。
1. 在转发面板点击 **"Pixhost"** 按钮（或其它支持批量的图床）。
2. 弹出选择框，勾选需要保留的截图。
3. 点击确认，脚本会在后台并发上传。
4. 上传完成后，目标站的简介框内会自动替换为新的图床链接。

### B. 制作对比图 (Comparison)
常用于发布官组作品或压制作品。
1. 打开 **图片处理** 大窗口。
2. 选择 **"制作对比图"** 标签页。
3. 输入 Source A 和 Source B 的图片链接。
4. 设置 Label（如 `Source vs Encode`）。
5. 点击生成，即可获得标准的 `[comparison]` 代码或居中对齐的图片组代码。

### C. Hostik 图床使用
Hostik 是一个非常优秀的图床，重构版对其进行了深度支持。
- **配置**: 在设置页面填入你的 Hostik API Key (如果有)。
- **使用**: 支持直接通过 API 上传（返回 JSON 结果），也支持通过桥接模式跳转上传。

> [!TIP]
> 所有的图片上传均在你的本地浏览器进行，图片流不经过任何第三方中转服务器，绝对安全保护隐私。
