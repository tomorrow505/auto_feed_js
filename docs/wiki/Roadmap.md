# 🗺️ 未来规划 (Roadmap)

Auto-Feed Refactor 的核心目标是成为 PT 圈最强大、最易用的发布与管理工具。以下是我们的中长期规划：

---

## 🟢 第一阶段：功能对齐 (Parity Phase)
**目标**: 100% 覆盖旧版脚本的所有核心功能。

- **深度站点适配**: 完成 PTP, HDB, CHD 的全字段、全模式对齐。
- **自动化增强**: 实现发布后的静默下载与本地辅种桥接。
- **历史记录管理**: 在面板中增加“最近转载”列表，方便快速重做或排查。

## 🟡 第二阶段：架构升级 (Architecture Phase)
**目标**: 解放生产力，让适配新站变得“零门槛”。

- **声明式适配器**: 开发一套 JSON 配置系统。适配一个新的 NexusPHP 站点只需一行配置，无需编写代码。
- **插件系统**: 允许开发者为不同站点编写“独立脚本”，通过 Hooks 注入到 Auto-Feed 主框架中。
- **多端同步**: 实验性支持导出配置到云端，在不同电脑间同步 API Keys 秘钥（加密存储）。

## 🔴 第三阶段：AI 赋能 (Intelligence Phase)
**目标**: 利用大模型能力，消灭“手动填表”。

- **智能简介生成**: 利用 LLM 自动识别 MediaInfo 内容，生成精美的 BBCode 简介。
- **自动改名建议**: 自动纠正不符合站点规则的种子标题。
- **智能查重**: 除了 IMDb 对比，通过语义识别判断资源是否已存在。

---

## 📈 长期愿景 (Vision)
我们希望 Auto-Feed 不仅仅是一个油猴脚本，而是一个 PT 生命周期管理框架，连接内容获取、元数据处理、种子清洗与远程控制。
